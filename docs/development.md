# Разработка

## Локальный запуск

```bash
python3 zonerec.py
python3 zonerec.py --start
python3 zonerec.py --stop
python3 zonerec.py --screenshot
```

Нужна активная X11-сессия и переменная `DISPLAY`. Перед запуском второй копии
трей-демона завершите первую, иначе процессы будут конкурировать за хоткеи.

## Базовые проверки

```bash
python3 -m py_compile zonerec.py
git diff --check
```

После изменений записи обязательно проверить все четыре режима звука: `none`,
`mic`, `system`, `both`. В тестовом ролике должны меняться изображение и курсор,
а длительность аудио и видео должна быть близкой.

Для диагностики файла полезны:

```bash
ffprobe -v error -show_entries stream=codec_name,duration,nb_frames recording.mp4
ffmpeg -v error -i recording.mp4 -map 0:v:0 -f framemd5 -
```

## Установка и сборка

`./install.sh` устанавливает зависимости и копирует приложение в `~/.local`.
Если зависимости уже установлены, для разработки достаточно заменить
`~/.local/bin/zonerec` и перезапустить процесс.

Debian-пакет собирается командой:

```bash
./build-deb.sh
```

Версия задаётся переменной окружения `VERSION`; результат появляется в `dist/`.

## Правила изменений

- не добавлять поддержку Wayland частично под видом X11-совместимости;
- сохранять корректное завершение `ffmpeg` через `SIGINT`;
- не накрывать область захвата GTK-окнами, даже прозрачными;
- при перестановке входов `ffmpeg` обновлять индексы `-map` и `amix`;
- не включать в коммит конфиги пользователя, записи, `.env`, `build/` и `dist/`.
