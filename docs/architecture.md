# Архитектура

ZoneRec намеренно собран в одном Python-файле. GTK обслуживает трей, диалоги и
выбор области, а внешние программы выполняют захват и системные действия.

## Поток записи

1. `do_start()` проверяет состояние и запускает `RegionSelector`.
2. Настройки определяют FPS, пресет и источники PulseAudio.
3. Экран и каждый аудиоисточник запускаются отдельными процессами `ffmpeg`.
   Благодаря этому медленная инициализация USB-микрофона не блокирует X11 и не
   создаёт очередь устаревших видеокадров.
4. Видео временно кодируется в H.264/MP4, каждый звук — в отдельный AAC/M4A.
5. PID процессов и пути временных файлов сохраняются в JSON state-файле.
6. При остановке процессы одновременно получают `SIGINT`. Аудиодорожки
   выравниваются по началу видео, дополняются тишиной в конце, смешиваются при
   необходимости и объединяются с видео без повторного кодирования изображения.
   Временные файлы удаляются.
7. Отдельный процесс показывает рамку из четырёх узких GTK-окон за пределами
   зоны. Одно прозрачное окно на всю область использовать нельзя: некоторые
   X11-композиторы возвращают под ним замороженное изображение.
8. `do_stop()` посылает `SIGINT`, чтобы `ffmpeg` корректно записал индекс MP4.

## Снимки экрана

`do_screenshot()` выбирает область, получает один кадр через `x11grab`, сохраняет
PNG/JPEG и по возможности передаёт изображение в GTK clipboard.

## Состояние и процессы

CLI и трей разделяют простой state-файл с PID `ffmpeg` и путём результата.
Состояние считается активным, только пока PID существует и не является zombie.
Рамка самостоятельно закрывается после исчезновения активной записи.

## Зависимости

- Python 3 и PyGObject;
- GTK 3, GDK Pixbuf, Keybinder и AppIndicator/Ayatana;
- `ffmpeg` с `x11grab`, PulseAudio и `libx264`;
- `pactl`, `xdg-open` и системные уведомления.
